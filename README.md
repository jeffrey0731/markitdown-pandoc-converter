# PDF/DOCX → Markdown 双引擎批量转换器 V3.0

一个基于 Tkinter 的桌面 GUI 工具，调用 **MarkItDown** 和 **Pandoc** 两大命令行引擎，将 PDF / Word（.docx）批量转换为 Markdown 文件，并针对"表格错位"这一常见痛点做了专门优化。

---

## 一、应用场景

- **文档知识库建设**：把大量 PDF 报告、Word 文档批量转换为 Markdown，导入 Obsidian、Notion、语雀等知识库工具
- **AI / RAG 预处理**：为大模型问答系统准备干净的 Markdown 语料，替代直接喂 PDF/DOCX 导致的排版混乱
- **扫描件 / OCR 文档处理**：对扫描版 PDF 自动识别提示，优先走 MarkItDown 的 OCR 表格解析
- **表格密集型 Word 文档**：财务报表、产品清单等含大量表格的 .docx，用 Pandoc 引擎减少行列错位
- **批量迁移**：老旧的 PDF/Word 文档存档，一次性拖拽多个文件批量转换为 Markdown 格式

不适用场景：非 PDF/DOCX 格式的转换（如 PPT、Excel、图片 OCR 等），当前版本仅聚焦 PDF/DOCX 两种输入格式。

---

## 二、运行环境

### 2.1 必需环境

| 项目 | 要求 |
|---|---|
| Python | 3.9 及以上（自带 tkinter，无需额外安装 GUI 库） |
| 操作系统 | Windows / macOS / Linux 均可运行 |
| 转换引擎 | 至少安装 `markitdown` 或 `pandoc` 其中一个命令行工具（建议两个都装，二选一时对应引擎会自动禁用） |

### 2.2 安装转换引擎

```bash
# 安装 MarkItDown（Python 包，自带命令行入口）
pip install "markitdown[all]"

# 安装 Pandoc（命令行工具，非 Python 包）
# Windows：
winget install --id JohnMacFarlane.Pandoc
# 或前往 https://pandoc.org/installing.html 下载安装包

# macOS：
brew install pandoc

# Linux（Debian/Ubuntu）：
sudo apt install pandoc
```

安装完成后，确认命令已加入系统环境变量（PATH），在终端中分别运行 `markitdown --help` 和 `pandoc --version` 能正常输出即可。

### 2.3 可选依赖（不装也能正常使用，仅缺少对应增强功能）

| 依赖包 | 作用 | 安装方式 |
|---|---|---|
| `tkinterdnd2` | 支持将文件直接拖拽进文件列表框 | `pip install tkinterdnd2` |
| `pdfplumber` | 自动检测 PDF 是否为扫描件，并给出引擎建议 | `pip install pdfplumber` |

未安装以上依赖时，程序会自动降级（用"添加文件"按钮代替拖拽、隐藏扫描检测选项），不会报错。

## 三、启动方式

```bash
python V3_0.py
```

首次启动会自动检测 `markitdown` 和 `pandoc` 是否可用，检测结果显示在"转换引擎"区域右上角。若某个工具未安装，对应的单选按钮会被禁用。

## 四、操作指导

### 4.1 选择转换引擎

- **MarkItDown**：适合 PDF、扫描件、OCR 场景，自带表格解析插件
- **Pandoc**：适合 DOCX、表格密集文档，行列错位问题更少
- **自动推荐**（勾选"按文件类型自动推荐引擎"）：程序会按每个文件的扩展名自动选择最合适的引擎（PDF → MarkItDown，DOCX → Pandoc），批量转换混合类型文件时无需手动切换

### 4.2 添加文件

有两种方式：
1. 点击**"添加文件..."**，在弹出的对话框中可多选 `.pdf` / `.docx` 文件
2. 直接将文件从资源管理器 / Finder 拖拽到文件列表框（需已安装 `tkinterdnd2`）

添加后的文件会显示在列表中，可通过"移除选中"或"清空列表"进行管理。

### 4.3 设置输出位置

- 默认勾选**"输出到源文件所在目录"**：每个文件转换后在原文件同目录生成同名 `.md`
- 取消勾选后可点击"浏览..."指定统一的输出目录，所有文件都会转换到该目录下

### 4.4 勾选选项

| 选项 | 说明 |
|---|---|
| 转换完成后自动打开输出目录 | 批量转换结束后自动弹出最后一个输出文件所在的文件夹 |
| 自动后处理 | 转换后自动统一换行符、清理连续空行，减少格式噪音 |
| 检测扫描版PDF | 转换前先判断 PDF 首页文字量，若判定为扫描件会在日志中提示优先使用 MarkItDown（需安装 `pdfplumber`） |

### 4.5 开始转换

点击**"▶ 开始批量转换"**，转换在后台线程中进行，不会卡住界面：

- 进度条 + 数字实时显示 `已完成/总数`
- 下方"日志"区域滚动打印每个文件的引擎选择、转换结果、报错详情
- 转换全部结束后弹窗汇总成功/失败数量

### 4.6 表格错位处理建议（转换后）

若转换出的 Markdown 表格仍有轻微错位，建议：
1. 用 VSCode 安装 **Markdown All in One** 插件，选中表格后按 `Shift+Alt+F` 一键格式化对齐
2. 扫描版 PDF 效果不理想时，尝试切换到另一引擎重新转换对比效果

### 4.7 配置记忆

程序退出时会自动保存以下设置到 `~/.md_converter_v3_config.json`，下次启动自动恢复：
- 上次使用的引擎
- 上次的输出目录设置
- 各功能开关状态（自动打开目录、后处理、扫描检测）
- 窗口大小

如需重置为默认设置，删除该配置文件即可。

## 五、常见问题

**Q: 提示"系统未找到命令行工具 markitdown/pandoc"？**
A: 说明该工具未安装或未加入系统 PATH。参考本文档"2.2 安装转换引擎"章节安装，安装后需重启终端 / 重新打开本程序。

**Q: 转换后表格还是错位怎么办？**
A: 尝试切换另一个引擎重新转换；或转换后用 VSCode + Markdown All in One 插件格式化；表格特别复杂的文档，人工微调是常见的最后一步。

**Q: 拖拽文件没反应？**
A: 拖拽功能依赖 `tkinterdnd2`，运行 `pip install tkinterdnd2` 后重新启动程序即可，未安装时请使用"添加文件"按钮。

**Q: 批量转换中途某个文件失败，会影响其他文件吗？**
A: 不会。程序按文件逐个处理，单个文件失败只会在日志中记录错误信息，不会中断整体批量流程。

**Q: 支持 .doc（旧版 Word）吗？**
A: 当前版本仅支持 `.pdf` 和 `.docx`，不支持旧版 `.doc` 格式，请先用 Word 另存为 `.docx` 后再转换。
