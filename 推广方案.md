## 先说人话：这工具是干什么的

**PDF/DOCX → Markdown 双引擎批量转换器**，一个 200 行出头的 Python GUI 小工具。

拖进去一堆 PDF、Word 文档，点一下"开始转换"，出来的是干干净净、表格基本不错位的 Markdown 文件。就这么简单。

没有付费墙，没有账号注册，没有"每天限免3次"，不联网也能跑——本质是给 `markitdown` 和 `pandoc` 这两个命令行神器套了一层"傻瓜式"的图形界面。

## 核心卖点，一图看懂

| 痛点 | 常见做法 | 这个工具怎么解决 |
|---|---|---|
| 表格错位 | 祈祷 | 双引擎实测调优参数（`--enable-table-plugin`/`--wrap=none`等），PDF走MarkItDown，DOCX走Pandoc，各自发挥所长 |

| 转换黑盒不知道哪错了 | 干瞪眼 | 实时日志窗口，每个文件成功/失败一目了然 |
| 中文路径乱码 | 抓狂 | Windows/macOS/Linux 全平台编码适配 |
| 每次都要重新设置 | 手动重复操作 | 配置自动记忆（引擎偏好、输出目录、窗口大小） |
| 转完还要手动整理格式 | 复制粘贴清理空行 | 自动后处理：统一换行符、清理多余空行 |

## 三行代码看懂它有多"轻"

没有 Electron，没有几百 MB 的安装包，没有依赖地狱：

```bash
pip install "markitdown[all]"   # 引擎1
brew install pandoc             # 引擎2（Windows/Linux见README）
python V3_0.py                  # 启动，完事
```

全部代码基于 Python 标准库 `tkinter`，跨平台零适配成本，代码总量控制在一个文件里，想改什么直接改，没有黑盒框架。

## 一分钟上手

1. 装好 `markitdown` 和/或 `pandoc`（任装一个即可跑，两个都装体验最佳）
2. `python V3_0.py` 启动
3. 把文件拖进窗口（或点"添加文件"）
4. 勾选"自动推荐引擎"，让程序自己判断用哪个
5. 点"开始批量转换"，泡杯咖啡的功夫，Markdown 全部生成好

详细图文教程见仓库 `README.md`。
